
# Mini Project 3 — Customer Shopping Behavior Analysis (SQL + Python + Power BI)

## 📌 Overview
This project analyzes a retail **customer shopping behavior dataset** (3,900 records, 18 columns) end-to-end, following a **Python → SQL → Power BI** pipeline:
1. **Python** — load the raw CSV, clean and engineer features, then push the cleaned DataFrame into a SQL database (PostgreSQL/MySQL) via SQLAlchemy.
2. **SQL** — 20 business questions answered directly against the database table using aggregate queries, subqueries, CTEs, and window functions.
3. **Power BI** — connects directly to the **SQL database** (not the raw CSV) to pull the cleaned `customer` table, then builds the interactive dashboard visualizing the results of all 20 questions.

**Data flow:** `customer_shopping_behavior.csv` → cleaned & transformed in **Python** → loaded into SQL table **`customer`** → queried in **SQL** → connected live in **Power BI** for visualization

## 🐍 Part 1: Python — Data Cleaning & Preparation
Notebook: `Customer_Shopping_Behavior_Analysis.ipynb`

Steps performed:
1. **Load data** with `pandas.read_csv()` and inspect with `.head()`, `.info()`, `.describe(include='all')`.
2. **Check for missing values** using `.isnull().sum()`.
3. **Impute missing `Review Rating`** values with the **median rating per Category** (`groupby('Category').transform(median)`), preserving category-level rating distribution instead of using a global median.
4. **Standardize column names** — lowercased and snake_cased (e.g. `Purchase Amount (USD)` → `purchase_amount`) for cleaner SQL/BI referencing.
5. **Feature engineering:**
   - `age_group` — binned `age` into **Young, Adult, Middle Aged, Senior Citizen** using `pd.cut()`.
   - `purchase_frequency_days` — mapped the categorical `frequency_of_purchases` (Weekly, Monthly, Quarterly, etc.) to a numeric day count for easier trend analysis.
6. **Data validation & deduplication** — discovered that `discount_applied` and `promo_code_used` were **100% identical columns** (verified with `(df['discount_applied'] == df['promo_code_used']).all()`), so `promo_code_used` was dropped as redundant.
7. **Load into database** — connected via `SQLAlchemy` (`create_engine`) to **PostgreSQL** (and alternatively MySQL via `pymysql`), and wrote the cleaned DataFrame to a `customer` table with `df.to_sql()` for downstream SQL analysis.

Buttons allow navigating from the home screen to each question's visual and back.

## 🧠 Key Business Insights
- **Male customers generate ~2x the revenue** of female customers ($158K vs $75K).
- **Non-subscribers outnumber subscribers** and contribute more *total* revenue, but subscribers show a **slightly higher average spend and estimated CLV** — subscription is correlated with modestly higher long-term value.
- **Loyal customers (79.9%) dominate the customer base**, indicating strong retention, though this also means the "New" customer acquisition pipeline (2.1%) is comparatively small.
- **Footwear, Clothing, and Accessories** have near-identical average order values (~$60); **Outerwear** trails slightly ($57).
- **Middle-aged and Adult customers** are the top revenue contributors, together driving well over half of total revenue.
- Revenue is **fairly evenly spread across seasons and locations** — no single season or state dominates, suggesting stable, non-seasonal demand.
- **PayPal** is the most-used payment method, but **Debit Card users spend the most per transaction on average** — a useful insight for targeted promotions.
- Discount usage is concentrated in specific items (**Hats, Sneakers, Coats**), suggesting these are the most price-sensitive/promotion-driven products.
